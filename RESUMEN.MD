## 2. Arquitectura y comunicaciĂłn (alto nivel)

- El servidor Express monta 3 grupos principales de rutas en [backend/src/app.js](backend/src/app.js):
  - `/api` â†’ [backend/src/routes/producto.routes.js](backend/src/routes/producto.routes.js) (productos)
  - `/api/paypal` â†’ [backend/src/router/paypal.router.js](backend/src/router/paypal.router.js) (lĂ­nea de pago)
  - `/api/venta` â†’ [backend/src/routes/venta.routes.js](backend/src/routes/venta.routes.js) (registro de ventas)

- Flujo de listado de productos:
  1. Frontend: `ProductoService.getAllFromApi()` solicita `GET http://localhost:3000/api/productos` (ver [src/app/Servicios/producto.service.ts](src/app/Servicios/producto.service.ts)).
  2. Backend: [backend/src/routes/producto.routes.js](backend/src/routes/producto.routes.js) enruta a [backend/src/controller/producto.controller.js](backend/src/controller/producto.controller.js) que ejecuta la consulta SQL en la DB a travĂ©s de [backend/src/config/db.js](backend/src/config/db.js).

- Flujo de pago (PayPal):
  1. Frontend `PaypalComponent` ( [src/app/Componentes/PayPal/paypal.ts](src/app/Componentes/PayPal/paypal.ts) ) carga el SDK de PayPal. Para obtener `client-id` intenta primero [GET /api/paypal/client-id] usando [src/app/Servicios/paypal.service.ts](src/app/Servicios/paypal.service.ts).
  2. Al iniciar pago, el frontend envĂ­a `POST /api/paypal/create-order` con `{ items, total }` (vĂ­a `PaypalService.crearOrden`).
  3. Backend [backend/src/router/paypal.router.js](backend/src/router/paypal.router.js) recibe `create-order` y lo pasa a [backend/src/controller/paypal.controller.js](backend/src/controller/paypal.controller.js) que llama a [backend/src/services/paypal.service.js](backend/src/services/paypal.service.js). Este mĂłdulo habla directamente con la API de PayPal (token OAuth â†’ crear orden).
  4. El backend retorna el `order.id` al frontend; el SDK de PayPal procesa el pago en cliente.
  5. Cuando PayPal confirma el pago, el frontend llama `POST /api/paypal/capture-order` (vĂ­a `PaypalService.capturarOrden`), el backend captura con PayPal, y el controller guarda la venta en la BD usando [backend/src/services/venta.service.js](backend/src/services/venta.service.js) (inserta `pedido` + `pedido_item` y actualiza stock en `producto`).

## 6. Flujo detallado de una compra (paso a paso, con archivos involucrados)

6.
